# JobLens 多角色上线风险评审

日期：2026-07-15

参与视角：产品负责人、应用安全工程师、后端工程师、SRE、测试负责人。

## 评审结论

项目满足当前匿名 MVP 的受控上线条件。模型成本、匿名滥用、敏感数据、依赖故障、线上可观测性和免费数据库备份均有明确防线。商业化前仍需轮换曾暴露的第三方密钥，并在 Render Free PostgreSQL 到期前完成数据库迁移或升级。

## 本轮决策

| 角色 | 风险判断 | 处理结果 |
|---|---|---|
| 产品负责人 | 免费匿名体验容易被批量滥用，但登录会显著增加转化摩擦 | 保留匿名体验，以 visitor/IP 日额度、Turnstile 和全局预算控制成本 |
| 安全工程师 | 监控 Token 与业务备份共用会扩大凭据泄露影响 | 新增独立 `BACKUP_TOKEN`，两个内部接口权限完全分离 |
| 数据负责人 | 业务恢复不需要 IP，备份复制 IP 不符合最小化原则 | 备份导出阶段移除所有 `ip_address` 字段 |
| SRE | 只监控 API 无法发现 Vercel 前端不可访问或发布错误 | 定时巡检增加首页 200、产品标题和 React 挂载节点检查 |
| 测试负责人 | 需要证明权限隔离而不只是检查无 Token | 增加监控 Token 无法调用备份接口、备份 Token 可通过认证的自动化测试 |

## 上线门禁

- 前后端构建、测试、依赖审计全部通过。
- Render API 健康检查中 PostgreSQL 与 Redis 均为可用。
- Vercel 首页和 API 同时通过生产巡检。
- 独立 Token 的业务快照完成加密、解密和结构校验并成功上传。
- `main` 与生产部署提交一致。

## 已接受的残余风险

- 浏览器匿名应用不能安全保存 HMAC 私钥，因此公开业务接口不采用伪请求签名；使用 Turnstile、分层配额和并发控制。
- GitHub Actions 告警依赖仓库通知设置，维护者仍需开启失败邮件通知。
- 应用级 JSON 快照不是 PostgreSQL PITR。进入付费或不可丢失数据阶段后必须使用带 PITR 的数据库。
- 当前免费 Render PostgreSQL 有到期时间，必须在到期前迁移。
