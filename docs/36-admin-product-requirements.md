# JobLens 管理后台产品需求文档

日期：2026-07-15
版本：P0 / v1.0

## 1. 产品目标

管理后台服务于产品负责人和运营者，每天回答五个问题：

1. 今天有多少真实使用，趋势是否异常？
2. 模型是否真的被调用，质量、耗时和成本是否可控？
3. 高风险岗位主要是什么类型，判断是否出现系统性偏差？
4. 用户提交了哪些纠错和面试反馈，哪些需要审核处理？
5. API、数据库、Redis、验证码和防滥用机制是否正常？

一期不建设组织、成员、复杂 RBAC、内容运营和模型配置中心。后台只供单一产品管理员使用，重点是数据可见、反馈可处理和风险可发现。

## 2. 用户与权限

| 用户 | 权限 | 一期实现 |
|---|---|---|
| 产品管理员 | 查看聚合指标、报告、反馈和系统状态；审核反馈 | 独立高熵 Admin Token |
| 普通访客 | 无后台权限 | 所有 `/api/admin/*` 请求返回 401 |

Admin Token 由服务端环境变量维护，前端不内置。管理员在 `/admin` 输入后只保存于当前标签页的 `sessionStorage`，关闭标签页即失效。

## 3. 信息架构

### 3.1 经营总览

- 时间范围：1 天、7 天、30 天。
- 核心指标：报告量、AI 模型调用率、高风险报告占比、平均风险分、模型估算成本、平均模型耗时、待审核反馈。
- 每日趋势：报告量、AI 调用量、高风险量。
- 风险等级分布：低、中、高、极高。
- 模型来源分布：provider/model 及调用量。
- 近期报告：时间、公司、岗位、风险等级、分数、模型来源。

### 3.2 分析质量

- 支持按报告 ID、公司、岗位搜索。
- 支持风险等级和分析来源筛选。
- 分页展示报告时间、岗位、风险分、置信度、模型、耗时、Token、成本。
- 查看详情：JD、HR 对话、风险类型、证据、缺失信息、追问建议和推荐结论。
- 不展示 visitor、IP、输入哈希等识别信息。

### 3.3 反馈审核

- 合并展示“报告纠错反馈”和“面试结果反馈”。
- 支持类型、审核状态筛选。
- 展示反馈正文、关联报告、岗位信息、风险标签和提交时间。
- 审核动作：通过、驳回、恢复待审核；可填写内部备注。
- 审核结果立即刷新待办数量，并记录安全审计事件。

### 3.4 系统与安全

- 依赖状态：PostgreSQL、Redis。
- AI 日预算：已用积分、总积分、使用率、重置时间。
- API 指标：总请求、成功率、4xx、5xx、限流次数、验证码要求/通过次数、AI 调用次数。
- 安全事件：最近事件类型、严重度、动作和时间；不展示 IP、visitor、user-agent。

## 4. 指标口径

| 指标 | 口径 |
|---|---|
| 报告量 | 时间范围内、未删除的 `JobReport` 数量 |
| AI 模型调用率 | provider 不为 `rule-based` 的报告数 / 报告量 |
| 高风险占比 | 风险等级为“高”或“极高”的报告数 / 报告量 |
| 平均风险分 | 报告 `overall_score` 平均值 |
| 模型估算成本 | 报告 `cost_estimate` 求和，仅供趋势参考 |
| 平均模型耗时 | 有 `latency_ms` 报告的平均值 |
| 待审核反馈 | 两类反馈中 `review_status=pending` 的未删除记录数 |
| API 成功率 | HTTP 状态码小于 400 的 API 日志数 / API 日志总数 |
| AI 调用量 | `ApiLog.ai_called=true` 的请求数 |

所有日期边界按用户选择范围滚动计算，界面明确显示“近 N 天”。无数据时展示 0 和空状态，不伪造趋势。

## 5. 交互与状态

- 登录失败明确提示凭据无效，不透露 Token 是否存在或长度规则。
- Token 失效或被轮换时自动退出后台并返回登录页。
- 切换时间范围只刷新数据区，不丢失登录态。
- 列表提供加载、空数据、错误和重试状态。
- 审核按钮提交期间禁用，成功后显示结果；失败保留用户填写的备注。
- 桌面端采用左侧导航和密集表格；移动端导航改为顶部横向页签，表格允许横向滚动。

## 6. 验收标准

- 未授权、错误 Token、监控 Token、备份 Token 均不能访问后台 API。
- 管理员可以在四个工作台看到上述完整数据，数据为空时页面仍可用。
- 时间筛选、搜索、状态筛选、分页和反馈审核均可实际操作。
- 后台接口不返回 IP、visitor、user-agent、输入哈希。
- 审核动作写入数据库并留下不含敏感信息的审计事件。
- 前后端构建、自动化测试、生产部署和线上回归全部通过。
