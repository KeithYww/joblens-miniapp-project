# 07 上线前检查

## 产品检查

| 项目 | 要求 | 状态 |
|---|---|---|
| MVP 范围 | 只保留 JD 检测、HR 分析、报告、追问、反馈 | 待确认 |
| 风险报告 | 必须展示证据、缺失信息、置信度 | 待确认 |
| 追问卡片 | 问题可复制，语言自然，不带攻击性 | 待确认 |
| 反馈提交 | 表单足够轻，选择题优先 | 待确认 |
| 分享内容 | 不直接展示企业负面定性 | 待确认 |

## 合规检查

| 项目 | 要求 | 状态 |
|---|---|---|
| 免责声明 | 报告页和分享页都要展示 | 待确认 |
| 隐私说明 | 检测前可见，设置页可查看完整说明 | 待确认 |
| 敏感词治理 | 避免“骗子、黑公司、实锤、曝光”等词 | 待确认 |
| 企业申诉 | 至少提供申诉邮箱或表单入口 | 待确认 |
| 误判反馈 | 每份报告都要有反馈入口 | 待确认 |
| 用户数据删除 | 提供删除检测记录的说明或入口 | 待确认 |

## 模型检查

| 项目 | 要求 | 状态 |
|---|---|---|
| JSON 输出稳定 | 字段结构固定，前端可直接渲染 | 待确认 |
| 无证据不高风险 | 模型不得在无证据情况下输出高风险 | 待确认 |
| 置信度 | 缺少 HR、工商或反馈时要降低置信度 | 待确认 |
| 强风险修正 | 押金、培训贷、扣证件等必须显著提示 | 待确认 |
| 追问生成 | 问题要具体，可直接发送给 HR | 待确认 |

## 技术检查

| 项目 | 要求 | 状态 |
|---|---|---|
| Web/H5 基础框架 | 页面路由、状态管理、接口调用完成 | 待确认 |
| 后端函数 | 风险检测、HR 分析、反馈提交可用 | 待确认 |
| 数据库 | reports、feedbacks、risk_terms 等表已创建 | 待确认 |
| 日志 | 记录接口失败、模型异常、用户反馈 | 待确认 |
| 限流 | 防止同一用户短时间高频调用 AI | 待确认 |
| 错误兜底 | AI 失败时返回友好提示 | 待确认 |

## 测试样本

上线前至少准备 50-100 条真实或脱敏岗位样本，覆盖：

- 管理岗包装销售岗
- 保险销售
- 培训贷
- 虚假高薪
- 外包伪装正式岗
- 无薪试岗
- 正常低风险岗位
- 正常销售岗位
- 初创公司正常岗位

## 灰度指标

| 指标 | 观察目标 |
|---|---|
| 检测完成率 | 不低于 50% |
| 报告命中感 | 不低于 60% |
| 追问复制率 | 不低于 25% |
| 二次检测率 | 不低于 20% |
| 面试反馈提交率 | 不低于 8% |
| 误判投诉率 | 可控，并能快速纠错 |

## 不建议上线的情况

出现以下情况时不建议公开上线：

- 风险报告经常只有分数，没有证据
- 模型容易把正常销售岗误判为骗局
- 用户不知道下一步该问 HR 什么
- 隐私说明和免责声明未完成
- 没有误判反馈入口
- 分享页容易引发企业投诉
