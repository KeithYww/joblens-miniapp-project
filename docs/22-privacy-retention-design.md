# 22 Privacy & Retention Design - P0 冻结版本

冻结日期：2026-07-13
版本：v1.0.0

## 说明

本文档冻结 JobLens Web/H5 MVP P0 阶段所有隐私保护与数据保留策略。所有前后端开发必须严格按照本策略实现数据收集、存储、脱敏、删除和日志记录。

隐私保护是 JobLens 的核心原则之一，原因：
- 用户提交 JD、HR 聊天记录、公司名称等信息涉及个人求职隐私
- 部分岗位信息可能包含联系方式、薪资范围等敏感信息
- AI 分析结果会发送至第三方大模型服务
- 必须避免用户信息被滥用、泄露或长期留存

## 数据分类与敏感等级

### 1. 高敏感数据

不得在日志、公开报告、统计分析中完整记录。

| 数据类型 | 说明 | 处理规则 |
|---|---|---|
| 身份证号 | 用户或他人身份证号 | 前端提示不得提交，后端检测到立即拦截并返回错误 |
| 完整手机号 | 11位手机号 | 前端提示不得提交，后端脱敏为 `138****5678` |
| 银行卡号 | 银行卡或支付账号 | 前端提示不得提交，后端检测到立即拦截 |
| 家庭住址 | 详细居住地址 | 前端提示不得提交，后端脱敏为城市级 |
| 真实姓名 | 用户或他人真实姓名 | 前端提示不得提交，后端脱敏为姓氏+先生/女士 |

### 2. 中敏感数据

可在数据库原文存储，但不得在日志、公开页面、统计分析中完整展示。

| 数据类型 | 说明 | 处理规则 |
|---|---|---|
| JD 原文 | 岗位描述完整文本 | 数据库原文存储，日志脱敏为前 100 字 + "...(已脱敏)" |
| HR 聊天原文 | HR 回复完整文本 | 数据库原文存储，日志脱敏为前 100 字 + "...(已脱敏)" |
| 公司名称 | 招聘公司名称 | 可正常展示，但不与企业工商信息强制关联 |
| 岗位名称 | 招聘岗位名称 | 可正常展示 |
| 薪资范围 | JD 中的薪资范围 | 可正常展示，不视为敏感数据 |

### 3. 低敏感数据

可正常记录、展示和分析。

| 数据类型 | 说明 | 处理规则 |
|---|---|---|
| 风险评分 | AI 生成的评分结果 | 可正常展示，不包含用户原文 |
| 证据列表 | AI 生成的风险证据 | 可正常展示，已脱敏处理 |
| 追问问题 | AI 生成的追问建议 | 可正常展示，不包含用户原文 |
| visitor_id | 前端匿名访问标识 | 可记录，不包含真实用户身份 |
| IP 地址 | 用户请求 IP | 可记录，用于限流和安全日志，不公开展示 |

## 数据保留策略

### 1. 报告数据保留

| 数据类型 | 保留时长 | 删除方式 | 说明 |
|---|---:|---|---|
| JD 原文 | 7 天 | 用户主动删除或自动过期 | 保留用于追问追加分析和缓存 |
| HR 聊天原文 | 7 天 | 用户主动删除或自动过期 | 同上 |
| 报告结果 | 30 天 | 用户主动删除或自动过期 | JSON 格式报告可保留较长用于用户查询历史 |
| 报告缓存 | 7 天 | Redis TTL 自动过期 | 重复输入可快速返回缓存报告 |

### 2. 反馈数据保留

| 数据类型 | 保留时长 | 删除方式 | 说明 |
|---|---:|---|---|
| 面试反馈 | 90 天 | 用户主动删除或自动过期 | 用于模型优化，需较长时间窗口 |
| 报告纠错反馈 | 90 天 | 用户主动删除或自动过期 | 同上 |
| 反馈审核状态 | 永久 | 不删除 | 用于统计和质量控制，不含原文 |

### 3. 日志数据保留

| 数据类型 | 保留时长 | 删除方式 | 说明 |
|---|---:|---|---|
| API 调用日志 | 30 天 | 自动过期 | 用于监控、审计和成本统计，不含敏感原文 |
| 安全事件日志 | 90 天 | 自动过期 | 用于安全分析和限流优化 |
| AI 调用日志 | 30 天 | 自动过期 | 用于成本统计和质量监控 |

## 用户删除权利

### 1. 删除入口

用户可以在以下页面主动删除数据：

| 页面 | 删除范围 | 说明 |
|---|---|---|
| 报告页 `/report/:id` | 该报告及关联数据 | 删除 JD 原文、HR 聊天原文、报告结果、HR 分析记录 |
| 反馈页 `/feedback` | 用户所有反馈 | 删除所有面试反馈和报告纠错反馈 |
| 隐私页 `/privacy` | 用户所有数据 | 一键删除所有数据，包括报告、反馈、日志记录 |

### 2. 删除流程

```text
用户点击"删除本次检测记录"
  ↓
前端确认弹窗："确定删除该报告及相关数据？删除后无法恢复。"
  ↓
用户确认
  ↓
POST DELETE /api/reports/:id
  ↓
后端立即设置 is_deleted=true, deleted_at=now()
  ↓
清除 Redis 缓存 report:{report_id}, report:hash:{input_hash}
  ↓
返回成功：{ "status": "deleted", "message": "该报告及相关数据已删除。" }
  ↓
前端跳转到首页并提示删除成功
```

### 3. 删除范围

删除报告时，必须同时删除或标记删除：

| 关联数据 | 处理方式 |
|---|---|
| JD 原文 | 标记删除，30天后物理删除 |
| HR 聊天原文 | 标记删除，30天后物理删除 |
| 报告结果 | 标记删除，30天后物理删除 |
| HR 分析记录 | 标记删除，30天后物理删除 |
| Redis 缓存 | 立即删除 |
| API 日志 | 保留但标记 report_id 已删除，不含原文 |
| 安全事件日志 | 保留，不含原文 |

## 日志脱敏规则

### 1. API 日志脱敏

日志表中不得记录以下内容：

| 禁止记录内容 | 原因 |
|---|---|
| JD 原文完整内容 | 用户隐私 |
| HR 聊天原文完整内容 | 用户隐私 |
| 用户提问完整内容 | 用户隐私 |
| HR 回复完整内容 | 第三方隐私 |
| 身份证号、手机号、银行卡号 | 高敏感数据 |
| 用户真实姓名 | 用户隐私 |

日志表只记录：

| 允许记录内容 | 示例 |
|---|---|
| 请求路径 | `/api/reports/detect` |
| 请求方法 | `POST` |
| visitor_id | `visitor_abc123` |
| IP 地址 | `192.168.1.1` |
| HTTP 状态码 | `200` |
| 错误码 | `VALIDATION_ERROR` |
| 错误消息（脱敏） | `JD 文本长度不符合要求` |
| AI Provider 名称 | `deepseek` |
| AI 模型名称 | `deepseek-v4-flash` |
| 耗时 ms | `1500` |
| token 数 | `input: 4000, output: 1200` |
| 成本估算 | `0.0064 元` |

### 2. 安全事件日志脱敏

安全事件日志只记录必要的元信息：

| 允许记录内容 | 示例 |
|---|---|
| 事件类型 | `rate_limit_hit` |
| 严重等级 | `medium` |
| visitor_id | `visitor_abc123` |
| IP 地址 | `192.168.1.1` |
| 请求路径 | `/api/reports/detect` |
| 请求 ID | `req_xyz789` |
| 处理动作 | `captcha_required` |

不得记录：
- 用户提交的具体内容
- 任何敏感文本片段

## AI 调用隐私边界

### 1. AI Provider 选择策略

| 场景 | 推荐策略 | 说明 |
|---|---|---|
| 开发/测试环境 | MockProvider + 脱敏样本 | 不提交真实用户数据到外部服务 |
| 内测阶段 | DeepSeek/SiliconFlow 低价付费模型 | 真实用户数据走隐私边界清晰的付费服务 |
| 公开 MVP | DeepSeek/SiliconFlow 低价付费模型主链路 | 免费模型只做降级或非敏感测试 |

**禁止：**
- 将真实用户 JD、HR 聊天原文提交至 Gemini Free 层（免费层内容会用于改进 Google 产品）
- 将真实用户数据提交至未经隐私条款审核的免费模型

### 2. 隐私说明文案

前端必须明确告知用户：

```text
检测页隐私提示：
请勿上传身份证、银行卡、完整手机号等敏感信息。检测结果仅供求职决策参考，不构成法律认定。

报告页隐私说明：
本结果基于你提供的信息生成，不构成法律认定或最终就业建议。你提交的岗位信息和 HR 聊天记录会被发送至第三方大模型服务用于生成风险报告。报告数据会在 7 天后自动删除，你也可以随时手动删除。

隐私页完整说明：
JobLens 不会永久保存你提交的 JD 原文和 HR 聊天记录。所有数据会在 7 天后自动删除，你也可以随时手动删除。我们不会将你的数据用于广告推荐或商业化用途。日志记录不包含你的原始文本内容。面试反馈会用于优化风险检测模型，但会在 90 天后自动删除。
```

## 统计分析脱敏

### 1. 可公开统计数据

| 统计维度 | 说明 | 脱敏要求 |
|---|---:|---|
| 检测总量 | 总检测次数 | 无需脱敏 |
| 风险等级分布 | 低/中/高/极高比例 | 无需脱敏 |
| 平均评分 | 平均 overall_score | 无需脱敏 |
| 热门平台 | 招聘平台使用频率 | 无需脱敏 |
| 热门岗位类型 | 岗位名称聚类统计 | 聚类后展示，不展示具体岗位名 |
| 热门公司 | 公司名称频率统计 | 频率超过 10 次的公司名可展示，低于 10 次的不展示 |

### 2. 内部统计数据（不公开）

| 统计维度 | 说明 | 脱敏要求 |
|---|---:|---|
| AI 成本统计 | Provider、模型、成本汇总 | 内部报表，不公开 |
| 限流命中率 | 限流触发频率 | 内部报表，不公开 |
| 误判反馈率 | 用户纠错反馈比例 | 内部报表，不公开 |
| 异常 IP 统计 | 异常行为 IP 列表 | 内部安全报表，不公开 |

## 隐私合规要点

### 1. 中国用户隐私保护

JobLens 需要遵守中国网络安全法、个人信息保护法等相关法规：

| 要求 | 实现措施 |
|---|---|
| 明确告知用户数据用途 | 前端隐私说明页明确说明 |
| 最小必要原则 | 只收集必要数据，不收集无关信息 |
| 用户同意原则 | 检测页明确告知，用户主动提交视为同意 |
| 用户删除权利 | 提供一键删除入口，7天内响应 |
| 数据本地化存储 | 数据库和日志存储在国内服务器 |
| 安全防护措施 | 限流、验证码、加密存储、访问控制 |

### 2. 第三方服务隐私条款

使用第三方 AI 服务时，必须：

| 要求 | 实现措施 |
|---|---|---|
| 选择隐私边界清晰的 Provider | DeepSeek、SiliconFlow 国内付费服务优先 |
| 避免免费层处理敏感数据 | 真实用户数据不走 Gemini Free、Groq Free |
| 前端明确告知第三方调用 | 隐私说明页明确说明发送至第三方大模型服务 |
| 用户可拒绝第三方调用 | 不提供检测功能，用户可选择不使用 |

## 后端实现要点

1. 所有敏感字段必须脱敏后才能写入日志
2. 数据库字段必须包含 `retention_until`、`is_deleted`、`deleted_at`
3. 用户删除请求必须立即响应，设置 `is_deleted=true`
4. 定时任务每日检查过期数据，标记删除
5. Redis 缓存必须设置合理的 TTL
6. AI 调用只选择隐私边界清晰的 Provider
7. 安全事件日志不得包含用户原文

---

## 前端实现要点

1. 检测页必须展示隐私提示，禁止上传身份证、银行卡、完整手机号
2. 报告页必须展示隐私说明，告知第三方 AI 调用和数据保留时长
3. 报告页必须提供"删除本次检测记录"入口
4. 隐私页必须提供一键删除所有数据入口
5. 删除操作必须二次确认，防止误删
6. 删除成功后跳转首页并提示
