# 06 小程序页面说明

## 页面列表

```text
/pages/index/index              首页
/pages/detect/detect            岗位检测
/pages/report/report            风险报告
/pages/ask-card/ask-card        追问卡片
/pages/feedback/feedback        面试反馈
/pages/privacy/privacy          隐私说明
/pages/disclaimer/disclaimer    免责声明
```

## 首页

### 目标

让用户快速理解产品，并立即开始检测。

### 核心文案

```text
粘贴岗位描述，识别信息不一致风险。
```

### 组件

- 产品名称：职镜 JobLens
- 主按钮：检测岗位风险
- 次按钮：分析 HR 回复
- 次按钮：提交面试反馈
- 隐私提示入口

### 注意

避免使用“诈骗公司、黑名单、曝光”等高风险词。

## 岗位检测页

### 目标

让用户低成本提交岗位信息。

### 字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 招聘平台 | 否 | BOSS、智联、猎聘、其他 |
| 公司名称 | 否 | 用户可手动输入 |
| 岗位名称 | 否 | 用户可手动输入 |
| JD 文本 | 是 | 岗位描述、职责、要求、薪资 |
| HR 聊天内容 | 否 | 可粘贴聊天记录 |

### 主按钮

```text
检测岗位风险
```

### 加载文案

```text
正在分析岗位标题、职责、薪资和风险话术...
```

## 风险报告页

### 目标

让用户理解风险原因，并知道下一步该问什么。

### 模块

- 综合风险分
- 风险等级
- 置信度
- 疑似真实岗位类型
- 命中风险信号
- 缺失关键信息
- 建议追问 HR 的问题
- 是否建议继续面试
- 免责声明
- 误判反馈入口

### 风险等级展示

| 分数 | 文案 |
|---:|---|
| 0-30 | 低风险，可正常沟通 |
| 31-60 | 中风险，建议补问关键信息 |
| 61-80 | 高风险，建议先电话确认 |
| 81-100 | 极高风险，谨慎继续 |

## 追问卡片页

### 目标

把风险报告转化为用户可以直接发给 HR 的问题。

### 功能

- 展示 3-8 个追问问题
- 一键复制单个问题
- 一键复制全部问题
- 粘贴 HR 回复继续分析

### 示例问题

```text
您好，我想确认一下：这个岗位前 1-3 个月是否有个人销售指标？
客户来源是公司提供，还是需要自己开发？
固定无责底薪是多少？
劳动合同签署主体是哪家公司？
```

## 面试反馈页

### 目标

沉淀真实面试反馈，形成长期数据壁垒。

### 字段

| 字段 | 类型 |
|---|---|
| 公司名称 | 文本 |
| 岗位名称 | 文本 |
| 招聘平台 | 单选 |
| JD 写的内容 | 文本 |
| 面试实际说的内容 | 文本 |
| 是否涉及销售 | 单选 |
| 是否涉及收费 | 单选 |
| 是否涉及培训贷 | 单选 |
| 是否涉及押金 | 单选 |
| 是否主体不一致 | 单选 |
| 是否推荐别人去 | 单选 |
| 是否匿名公开 | 开关 |

### 提交后文案

```text
已匿名提交。审核后将用于优化岗位风险判断，帮助更多求职者少跑无效面试。
```

## 隐私说明页

必须说明：

- 用户输入的 JD 和聊天记录用于生成风险报告
- 聊天记录原文不会公开展示
- 用户可以请求删除自己的检测记录
- 匿名反馈审核后才会用于统计
- 不建议上传身份证、银行卡、完整手机号等敏感信息

## 免责声明页

必须说明：

- 风险报告仅供求职决策参考
- 产品不对企业或岗位作法律定性
- 工商信息和用户反馈可能存在滞后或不完整
- 用户应结合实际沟通、劳动合同和官方渠道判断
