# 13 Web/H5 前端实现方案

## 推荐方案

首版推荐使用：

```text
Vite + React + TypeScript + React Router + CSS Modules/Tailwind
```

选择理由：

- 启动快，工程复杂度低
- 适合工具型 H5
- 移动端和桌面端都容易适配
- 后期可以迁移到 Next.js 做 SEO 和内容页

## 页面路由

| 路由 | 页面 | 说明 |
|---|---|---|
| `/` | 首页 | 产品说明与主入口 |
| `/detect` | 岗位检测 | 粘贴 JD / HR 聊天 |
| `/report/:id` | 风险报告 | 展示检测结果 |
| `/ask-card/:id` | 追问卡片 | 复制追问问题 |
| `/feedback` | 面试反馈 | 匿名反馈 |
| `/privacy` | 隐私说明 | 数据处理说明 |
| `/disclaimer` | 免责声明 | 风险提示边界 |

P1：

| 路由 | 页面 | 说明 |
|---|---|---|
| `/ocr` | 截图上传 | 上传岗位截图 |
| `/ocr/confirm/:id` | OCR 确认 | 用户确认识别字段 |

## 前端状态设计

首版不需要复杂状态管理，建议：

- 表单状态：React Hook Form 或组件内状态
- 报告详情：接口读取
- 临时检测草稿：`sessionStorage`
- 全局提示：轻量 Toast
- 不做用户登录，不做长期本地历史

## 表单校验

### 岗位检测

| 字段 | 校验 |
|---|---|
| JD 文本 | 必填，建议 50-8000 字 |
| HR 聊天 | 选填，建议不超过 8000 字 |
| 公司名称 | 选填，不超过 80 字 |
| 岗位名称 | 选填，不超过 80 字 |
| 招聘平台 | 选填，枚举或其他 |

### 面试反馈

反馈表单必须短，选择题优先。

必填字段：

- 公司名称
- 岗位名称
- 面试实际内容
- 是否涉及销售
- 是否涉及收费/押金/培训贷
- 是否推荐别人去

## 移动端适配

设计原则：

- 主要按钮固定在底部或跟随内容结尾
- 输入框高度适中，支持自动增高
- 报告页首屏必须展示风险等级和一句话建议
- 追问问题支持一键复制
- 长报告拆成卡片，避免整页大段文字

断点建议：

```text
< 768px：移动 H5 单列
>= 768px：平板/桌面双列
>= 1200px：桌面宽屏，左侧流程导航 + 主内容
```

## P1 截图上传前端方案

Web/H5 使用：

```html
<input type="file" accept="image/*" multiple>
```

限制：

- 最多 3 张
- 单张不超过 5MB
- 允许 jpg、jpeg、png、webp
- 上传前本地预览
- 可删除已选图片

流程：

```text
选择图片
  ↓
本地校验
  ↓
上传到 /api/ocr/extract
  ↓
展示识别结果确认页
  ↓
确认后调用 /api/reports/detect
```

## 错误兜底

| 场景 | 前端处理 |
|---|---|
| AI 接口失败 | 提示稍后重试，保留用户输入 |
| 返回 JSON 不完整 | 展示兜底文案，并允许重新检测 |
| 文本过短 | 提示补充岗位职责和薪资 |
| 图片识别失败 | 提示用户粘贴文本 |
| 复制失败 | 展示文本框供手动复制 |
| 请求限流 | 提示稍后再试 |

## 隐私提示

在检测页输入框下方展示：

```text
请勿上传身份证、银行卡、完整手机号等敏感信息。检测结果仅供求职决策参考。
```

在报告页展示：

```text
本结果基于你提供的信息生成，不构成法律认定或最终就业建议。
```
