# 01 项目简报

## 项目名称

职镜 JobLens

## 产品一句话

粘贴岗位 JD 或 HR 回复，识别岗位信息不一致、销售包装、薪资不透明、主体不清等求职风险，并生成面试前追问问题。

## 背景

求职者在招聘平台上经常遇到岗位名称和实际工作内容不一致的情况，例如“管理岗”“储备干部”“综合管理”实际是保险销售、电话销售、地推或培训贷导流。用户真正损失的不只是金钱，也包括通勤时间、准备成本、情绪消耗和对招聘平台的信任。

## 首版目标

第一版不做招聘平台，也不做公司点评社区，而是做一个轻量的 Web/H5 工具：

- 帮用户在面试前识别岗位风险
- 帮用户知道应该追问 HR 什么
- 通过匿名反馈沉淀真实面试信息
- 为后续公司岗位画像和风险库积累数据

## 非目标

第一版不做以下内容：

- 不做全网岗位搜索
- 不做大规模岗位爬虫
- 不做公司黑名单
- 不做公开曝光企业
- 不做复杂社区
- 不做付费会员
- 不做法律定性

## 首批目标用户

| 用户 | 典型痛点 |
|---|---|
| 应届生 | 分不清管培生、储备干部、培训贷、付费内推 |
| 转行者 | 对行业不熟，容易被“跨行业培养管理人才”吸引 |
| 空窗期求职者 | 急于找工作，降低筛选标准 |
| 技术/职能转管理人群 | 容易被伪管理岗误导 |
| 被包装岗位坑过的人 | 有强烈分享和避坑需求 |

## 首版核心闭环

```text
粘贴 JD
  ↓
生成风险报告
  ↓
复制追问问题
  ↓
粘贴 HR 回复继续分析
  ↓
面试后匿名反馈
```

## 核心判断

产品价值不在于“告诉用户某家公司一定有问题”，而在于：

- 告诉用户哪些信息不透明
- 告诉用户哪些话术存在包装风险
- 告诉用户哪些问题必须在面试前问清楚
- 帮用户减少无效面试
