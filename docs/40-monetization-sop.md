## 📊 关键指标追踪

### 北极星指标与口径

北极星指标为**每周完成有效风险报告的独立用户数（WACR）**。一次“有效报告”必须成功生成结果页，测试流量、管理员流量和重复刷新不计入。

| 指标 | 统一计算口径 |
|------|--------------|
| 激活率 | 完成首份有效报告的独立访客 ÷ 落地页独立访客 |
| 注册转化率 | 完成注册的独立访客 ÷ 落地页独立访客 |
| 付费转化率 | 首次支付成功用户 ÷ 到达付费墙的独立用户 |
| 7 日留存 | 首次生成报告后第 7 天（±1 天）再次生成报告的用户 ÷ 对应新增用户 |
| CAC | 某渠道可归因推广成本 ÷ 该渠道新增付费用户 |
| ARPPU | 当月实收订阅收入 ÷ 当月付费用户数 |
| 毛利率 |（实收收入－支付手续费－AI 增量成本－退款）÷ 实收收入 |

所有推广链接必须携带 `utm_source`、`utm_medium`、`utm_campaign` 和内容编号 `utm_content`。每周一固定导出上周数据，避免不同平台口径混用。

### 必埋事件

| 事件 | 触发时机 | 必要属性 |
|------|----------|----------|
| `landing_view` | 打开落地页 | source、campaign、content |
| `analysis_start` | 提交检测 | input_type、is_logged_in |
| `report_success` | 报告成功展示 | report_id、risk_level、analysis_source |
| `paywall_view` | 看到升级入口 | trigger、plan |
| `checkout_start` | 发起下单 | plan、amount、channel |
| `payment_success` | 服务端确认支付 | order_id、plan、amount、channel |
| `refund_success` | 退款完成 | order_id、amount、reason |
| `share_click` | 点击分享 | channel、report_id |

事件中禁止上传岗位原文、聊天原文、手机号、支付凭证或其他个人信息。`payment_success` 必须以服务端验签结果为准。

### 阶段门槛（达到后再扩张）

| 关口 | 最低通过条件 | 未通过动作 |
|------|--------------|------------|
| 基建关（第 2 周） | 支付或付费意向链路端到端通过；关键事件可查；退款/客服流程已演练 | 暂停投放，继续修链路 |
| 需求关（第 4 周） | ≥30 名真实目标用户完成报告；≥10 次访谈；≥5 个明确付费意向 | 收窄人群或重写价值主张 |
| 渠道关（第 8 周） | 至少 1 个渠道连续 2 周带来 ≥20 个激活用户/周 | 停止低效日更，改做 2 个新渠道实验 |
| 付费关（第 12 周） | ≥10 笔真实支付，退款率 <10%，单个渠道 CAC 不高于首月毛利 | 调整权益、价格或付费触发点 |
| 扩张关（第 16 周） | 付费转化与留存连续 4 周不下降，客服可在 24h 内响应 | 不启动企业版，优先修留存 |

### 内容实验卡（每次发布前填写）

```text
实验编号：CH-YYYYMMDD-01
目标人群：例如“应届生，拿到第一份销售岗 offer”
用户痛点：担心薪资结构和岗位包装
单一假设：案例拆解比功能演示带来更高的报告完成率
内容钩子：一句话
CTA：上传一份 JD 完成风险检测
渠道与 UTM：
观察窗口：发布后 72 小时
成功标准：≥20 次落地页访问且激活率 ≥15%（样本不足只记录，不下结论）
结果：曝光 / 点击 / 激活 / 付费意向 / 支付
决策：加码 / 修改 / 停止
```

每周最多同时运行 3 个核心假设；每个实验只改一个主要变量。自然流量样本很小时用访谈证据辅助判断，不因一次爆文直接扩大预算。
