# JobLens 仿真数据回归测试报告

> 执行日期：2026-07-17
> 样本性质：全部为程序构造的虚构岗位，不包含真实用户、公司或个人信息。

## 测试范围

第一轮建立 50 条中文确定性仿真样本，覆盖：

| 场景 | 数量 |
|------|----:|
| 正常透明销售岗 | 8 |
| 管理岗包装销售岗 | 8 |
| 培训贷/贷款分期 | 6 |
| 入职押金/保证金 | 6 |
| 扣留证件 | 4 |
| 无薪试岗 | 4 |
| 拉亲友资源 | 4 |
| 信息不足 | 5 |
| 正常专业岗位 | 5 |
| **合计** | **50** |

第二轮新增 20 条英文仿真样本：5 条透明销售岗、4 条 management trainee 包装、4 条 training loan、3 条 upfront fee/security deposit、2 条扣留护照和 2 条信息不足岗位。

每条输出同时检查风险分、风险等级、风险类型、岗位预测、禁止误判、敏感定性表达和完整 API Schema。门禁要求为样本通过率不低于 95%。

## 首轮结果与缺陷

| 轮次 | 通过率 | 发现的问题 |
|------|-------:|------------|
| 1 | 2% | `Math.random()` 生成的报告 ID 长度不稳定，绝大多数输出不符合 `rep_[a-z0-9]{12}` 契约 |
| 2 | 52% | 强风险证据文本短于 Schema 的最小 10 字要求 |
| 3 | 72% | 无薪试岗、拉亲友资源虽被命中，但风险分只有 50，低于业务规则要求 |
| 最终 | 100% | 50/50 样本通过 |

## 已修复内容

1. 使用 6 字节加密随机数生成固定 12 位十六进制 ID，报告和 HR 分析 ID 均稳定满足契约；
2. 强风险证据改为完整、可解释且符合 Schema 长度限制的描述；
3. 无薪试岗风险分最低为 75；
4. 拉亲友资源风险分最低为 70；
5. 默认测试命令加入仿真回归门禁，并提供独立命令 `npm run test:regression`。

## 第二轮英文验证

英文样本首轮仅 2/20 通过，通过率 10%。原因是前端和报告本地化虽然支持英文，但模型不可用时启用的本地规则引擎只识别中文信号。

修复后增加以下双语规则：

- `training loan`、`loan installments`；
- `upfront fee`、`security deposit`；
- `retain/hold passport`、`retain diploma`；
- `unpaid trial`、`recruit friends and family`；
- `management trainee` 与 field marketing、client acquisition、lead generation、performance targets 的联合判断；
- 对明确写出 fixed base salary、commission 和销售目标的正常销售岗实施防误判。

第二轮最终结果为 20/20 通过。中文 50 条与英文 20 条共同组成 70 条仿真回归门禁。

## 最终验证

- 中文仿真回归：50/50 通过，通过率 100%；
- 英文仿真回归：20/20 通过，通过率 100%；
- 后端完整测试：29 通过、0 失败、1 跳过；
- 后端 TypeScript 构建：通过；
- 跳过项为依赖数据库的管理后台聚合测试；本机 PostgreSQL 未运行，测试日志中的数据库连接失败为预期的降级/安全路径验证。

## 结论与边界

本轮证明规则引擎对已定义的中文典型模式能够稳定回归，并发现、修复了三个此前未覆盖的契约/评分问题。它不能证明真实市场准确率，也不能替代以下测试：

- 50 条以上脱敏真实岗位的人工标注回归；
- 英文岗位、海外招聘诈骗话术回归；
- 真实大模型 Provider 的非确定性质量评估；
- 浏览器端完整用户流程和截图 OCR 视觉回归；
- 有 PostgreSQL/Redis 的完整集成测试。

下一阶段应优先引入脱敏真实样本，并分别统计高风险召回率、正常岗位误报率和人工复核一致率。
