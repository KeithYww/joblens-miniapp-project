# 02 MVP 范围

## 首版原则

第一版只验证一个问题：用户是否愿意把岗位信息交给产品检测，并认为检测结果能帮助自己决定是否继续面试。

## 必做功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| JD 文本检测 | 用户粘贴岗位描述，系统输出风险报告 | P0 |
| HR 回复分析 | 用户粘贴 HR 回复，系统判断是否回避核心问题 | P0 |
| 风险报告页 | 展示风险分、等级、证据、缺失信息、建议 | P0 |
| 追问卡片 | 生成可复制给 HR 的追问问题 | P0 |
| 匿名反馈 | 用户提交面试实际情况，沉淀风险样本 | P0 |
| 隐私说明 | 告知用户数据如何使用、保存和删除 | P0 |
| 免责声明 | 告知风险报告仅供求职决策参考 | P0 |

## 可选功能

| 功能 | 说明 | 是否进入首版 |
|---|---|---|
| 检测历史 | 展示用户最近检测过的岗位 | 可做简单版 |
| 分享卡片 | 分享“需确认问题”，不展示企业负面定性 | 可做 |
| 避坑词典 | 展示高频招聘话术解释 | 可后置 |
| 上传岗位截图识别 | 用户上传岗位页或 HR 聊天截图，OCR 后回填岗位信息 | P1，可在文本检测跑通后加入 |

## 暂缓功能

| 功能 | 暂缓原因 |
|---|---|
| 工商 API 自动查询 | 成本和合规复杂度较高，MVP 可先用主体追问替代 |
| 岗位链接自动解析 | 不同平台限制多，首版先让用户复制 JD |
| 全网岗位抓取 | 协议、反爬、隐私和法律风险高 |
| 公司黑名单 | 容易产生争议和审核风险 |
| 社区广场 | 内容治理成本高，容易变成吐槽区 |
| 企业认证 | 太早，会影响产品中立性 |
| 付费会员 | 早期强收费会降低传播和数据积累 |

## 页面范围

首版页面：

- 首页
- 岗位检测页
- 风险报告页
- 追问卡片页
- 面试反馈页
- 隐私说明页
- 免责声明页

## 首版验收标准

| 指标 | 目标 |
|---|---|
| 检测完成率 | 用户进入检测页后，完成一次检测的比例不低于 50% |
| 报告命中感 | 用户主观反馈“有帮助/说中了”的比例不低于 60% |
| 追问复制率 | 用户复制追问问题的比例不低于 25% |
| 二次检测率 | 7 日内再次检测岗位的用户比例不低于 20% |
| 反馈提交率 | 完成检测后提交面试反馈的比例不低于 8% |

## 首版上线判断

如果用户只把它当成一次性娱乐工具，不愿意粘贴真实 JD、不愿意复制追问、不愿意反馈面试结果，就不应继续投入工商查询、社区和全平台数据接入。
