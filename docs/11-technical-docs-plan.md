# 11 技术文档规划

## 目标

本目录用于指导 `职镜 JobLens` Web/H5 首版工程实现。技术文档需要让前端、后端、算法/AI、测试和部署人员都能明确边界，避免在 MVP 阶段扩大范围。

## 技术路线总判断

首版技术方案建议：

```text
Web/H5 前端
  ↓
后端 API
  ↓
AI 风险分析服务
  ↓
规则校验与风险评分
  ↓
数据库持久化
```

P1 后再接：

```text
截图上传
  ↓
自建 OCR 服务
  ↓
岗位字段结构化
  ↓
用户确认
  ↓
风险检测
```

## 首版技术边界

### 必须实现

- Web/H5 响应式页面
- JD 文本检测
- HR 回复分析
- 风险报告生成
- 追问问题复制
- 匿名反馈提交
- 报告误判反馈
- 隐私说明与免责声明

### 预留但不阻塞

- 截图 OCR
- 工商 API 查询
- 用户登录
- 历史记录
- 公司画像
- 管理后台

### 明确不做

- 全网岗位抓取
- 企业黑名单
- 公开社区
- 复杂会员体系
- App
- 小程序

## 技术验收原则

首版上线前必须满足：

- AI 输出 JSON 字段稳定
- 风险报告有证据、有缺失信息、有置信度
- 高风险结论必须有证据片段
- 前端在移动端浏览器可正常使用
- 用户可以不登录完成一次检测
- 用户可以删除或反馈检测结果
- 后端能限制高频调用，避免 AI 成本失控

## 推荐开发顺序

1. 建立 Web/H5 前端工程
2. 实现静态页面和交互原型
3. 实现后端 API 骨架
4. 接入 AI 风险分析结构化输出
5. 落库 `job_reports` 和 `feedbacks`
6. 加入规则校验和强风险修正
7. 加入隐私、免责声明和误判反馈
8. 用 50-100 条岗位样本测试
9. 小流量灰度
