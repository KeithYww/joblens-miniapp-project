# 职镜 JobLens 页面流程

## 页面范围

首版页面与 PRD 保持一致：

| 路由 | 页面 | 主要任务 |
|---|---|---|
| `/pages/index/index` | 首页 | 解释产品价值，进入检测、HR 分析或反馈 |
| `/pages/detect/detect` | 岗位检测 | 提交 JD、岗位信息和可选 HR 聊天内容 |
| `/pages/report/report` | 风险报告 | 展示风险分、证据、缺失信息和建议动作 |
| `/pages/ask-card/ask-card` | 追问卡片 | 复制单个或全部追问问题 |
| `/pages/feedback/feedback` | 面试反馈 | 匿名提交面试实际情况 |
| `/pages/privacy/privacy` | 隐私说明 | 说明数据使用、保存、删除和敏感信息限制 |
| `/pages/disclaimer/disclaimer` | 免责声明 | 说明报告边界和用户判断责任 |

## 主流程

```text
用户进入首页
  ↓
点击“检测岗位风险”
  ↓
粘贴 JD，填写可选岗位信息
  ↓
点击“检测岗位风险”
  ↓
系统生成风险报告
  ↓
用户阅读风险摘要与命中证据
  ↓
复制追问问题
  ↓
粘贴 HR 回复继续分析
  ↓
面试后提交匿名反馈
```

主流程的设计重点是让用户从“看到风险”进入“问清风险”。报告页不是终点，追问卡片和 HR 回复分析才是首版留存与价值验证的关键动作。

## 首页流程

```text
首页
  ├─ 主按钮：检测岗位风险 → 岗位检测页
  ├─ 次按钮：分析 HR 回复 → 岗位检测页（展开 HR 聊天输入）
  ├─ 次按钮：提交面试反馈 → 面试反馈页
  ├─ 隐私说明入口 → 隐私说明页
  └─ 免责声明入口 → 免责声明页
```

首页不展示复杂模型介绍。用户只需要理解三件事：粘贴内容、识别风险、得到可追问的问题。

## 岗位检测流程

```text
岗位检测页
  ↓
输入 JD 文本
  ├─ 少于 50 字：按钮可点击但提示内容可能不足
  ├─ 空内容：按钮不可提交
  └─ 包含敏感信息：显示删除敏感信息提示
  ↓
点击检测
  ↓
分析中
  ├─ 成功：进入风险报告页
  ├─ 失败：停留本页并保留输入
  └─ 超时：提示稍后重试，可复制已输入内容
```

字段优先级：

| 字段 | 页面位置 | 必填 | 交互规则 |
|---|---|---|---|
| JD 文本 | 首屏主输入区 | 是 | 支持粘贴长文本，显示字数 |
| HR 聊天内容 | 折叠区 | 否 | 从“分析 HR 回复”进入时默认展开 |
| 招聘平台 | 次级字段 | 否 | 使用标签选择，允许“其他” |
| 公司名称 | 次级字段 | 否 | 用户可跳过 |
| 岗位名称 | 次级字段 | 否 | 用户可跳过 |

## 风险报告流程

```text
风险报告页
  ↓
查看风险摘要
  ├─ 低风险：可正常沟通，仍提示补问缺失信息
  ├─ 中风险：建议补问关键信息
  ├─ 高风险：建议先电话确认
  └─ 极高风险：谨慎继续，涉及收费/押金/贷款时建议停止流程
  ↓
查看命中证据
  ↓
查看缺失信息
  ↓
点击“生成追问卡片”
  ↓
追问卡片页
```

报告页首屏需要同时出现：

- 综合风险分
- 风险等级
- 置信度
- 疑似真实岗位类型
- 一句话建议
- “复制追问问题”主按钮

子分说明可放在报告中后段。冷启动阶段没有工商信息或用户反馈时，显示“暂未接入/暂无反馈样本”，不得把缺失数据解释为低风险。

## 追问卡片流程

```text
追问卡片页
  ├─ 复制单个问题
  ├─ 复制全部问题
  ├─ 粘贴 HR 回复继续分析 → 岗位检测页（HR 输入展开）
  └─ 面试后反馈 → 面试反馈页
```

复制成功后显示轻提示“已复制，可粘贴给 HR”。如果浏览器复制权限失败，展示问题文本并提示用户手动复制。

## HR 回复分析流程

```text
入口：首页“分析 HR 回复”或追问卡片“粘贴 HR 回复继续分析”
  ↓
检测页展开 HR 聊天内容输入区
  ↓
用户粘贴 HR 回复
  ↓
系统判断是否回避核心问题
  ↓
风险报告页更新 HR 话术风险分和新的追问建议
```

HR 回复分析强调“是否回答了关键问题”，不评价 HR 个人态度。高风险信号包括回避底薪、销售指标、客户来源、合同主体、收费项目等。

## 面试反馈流程

```text
面试反馈页
  ↓
填写公司、岗位、平台
  ↓
填写 JD 写法和面试实际说法
  ↓
选择是否涉及销售、收费、培训贷、押金、主体不一致
  ↓
选择是否推荐别人去
  ↓
选择是否匿名公开
  ↓
提交成功
  ↓
展示感谢文案和“继续检测其他岗位”
```

反馈页提交后文案：

```text
已匿名提交。审核后将用于优化岗位风险判断，帮助更多求职者少跑无效面试。
```

## 异常与边界流程

| 场景 | 页面 | 处理方式 |
|---|---|---|
| JD 内容为空 | 岗位检测 | 禁用提交按钮，提示“请先粘贴岗位描述” |
| JD 内容过短 | 岗位检测 | 可提交，但提示“信息较少，报告置信度可能偏低” |
| 用户输入敏感信息 | 岗位检测、反馈 | 提示删除身份证、银行卡、完整手机号等信息 |
| 模型返回无证据高风险 | 风险报告 | 降级为“信息不足”，要求补充证据后再给高风险 |
| 工商信息缺失 | 风险报告 | 标注“暂未接入工商信息”，不计入低风险 |
| 复制失败 | 追问卡片 | 展示可手动选中的问题文本 |
| 反馈提交失败 | 面试反馈 | 保留表单内容，允许再次提交 |

## Web/H5 页面组织

Web/H5 原型采用一个 HTML 文件展示多页面状态：

```text
顶部：产品名与页面锚点导航
主体：响应式屏幕画布
  ├─ 首页
  ├─ 岗位检测
  ├─ 风险报告
  ├─ 追问卡片
  ├─ 面试反馈
  ├─ 隐私说明
  └─ 免责声明
侧栏：页面说明与关键交互
```

在真实 H5 中，页面可以拆成独立路由；设计原型把它们放在同一文件，便于评审、走查和开发对照。

## 页面验收口径

| 页面 | 可验收表现 |
|---|---|
| 首页 | 用户 5 秒内能理解产品用途，并看到“检测岗位风险”主按钮 |
| 岗位检测 | 用户能直接粘贴 JD，知道哪些字段可跳过 |
| 风险报告 | 首屏能看到风险分、等级、证据入口和下一步建议 |
| 追问卡片 | 每个问题都能独立复制，也能复制全部 |
| 面试反馈 | 表单字段覆盖 PRD 要求，默认匿名，说明审核后使用 |
| 隐私说明 | 明确聊天记录原文不会公开展示，可请求删除 |
| 免责声明 | 明确报告仅供求职决策参考，不作法律定性 |
