# 职镜 JobLens UI 设计说明

## 设计目标

职镜 JobLens 的首版界面服务于一个核心动作：用户把岗位 JD 或 HR 回复交给产品检测，产品用清晰、克制、可解释的方式帮助用户判断是否继续沟通。界面不承担“曝光企业”或“给公司定性”的角色，而是把风险拆成证据、缺失信息和面试前需要问清的问题。

设计需要同时兼容微信小程序、移动 H5 和桌面 Web 预览。移动端以单手操作、短时间完成检测为主；Web 端以更宽的内容阅读、报告对照和设计评审为主。

## 产品气质

| 维度 | 设计取向 | 说明 |
|---|---|---|
| 信任感 | 清晰、理性、克制 | 避免“黑名单”“曝光”“诈骗”等高风险表达 |
| 工具感 | 直接、低负担 | 首页到检测页不超过 1 次主点击 |
| 安全感 | 明确隐私边界 | 输入区附近持续提示不要上传身份证、银行卡、完整手机号 |
| 解释性 | 证据优先 | 风险报告以“命中片段 + 为什么要问清楚”替代主观判断 |
| 行动感 | 可复制追问 | 报告结果必须能转化为用户下一句发给 HR 的问题 |

## 信息架构

首版只保留一条主路径：

```text
首页
  ├─ 岗位检测
  │    └─ 风险报告
  │         ├─ 追问卡片
  │         ├─ 粘贴 HR 回复继续分析
  │         └─ 面试后匿名反馈
  ├─ 分析 HR 回复
  ├─ 提交面试反馈
  ├─ 隐私说明
  └─ 免责声明
```

小程序端采用底部安全区内的单列布局；H5/Web 端保留同一内容结构，在宽屏下增加左侧流程导航和右侧报告摘要，不改变核心表单字段。

## 视觉系统

### 色彩

| Token | 建议色值 | 用途 |
|---|---:|---|
| `--bg` | `#F6F3EC` | 页面背景，带轻微暖色，减少焦虑感 |
| `--surface` | `#FFFFFF` | 卡片、表单、报告模块 |
| `--surface-soft` | `#F0ECE2` | 次级卡片、提示条 |
| `--ink` | `#1F2933` | 正文主文字 |
| `--muted` | `#667085` | 说明文字、辅助信息 |
| `--rule` | `#E2DDD2` | 分割线、输入框边框 |
| `--brand` | `#2F6B5F` | 主按钮、品牌识别、低风险正向提示 |
| `--brand-strong` | `#1F4D43` | 主按钮按下态、标题重点 |
| `--warning` | `#B7791F` | 中风险提示 |
| `--danger` | `#B42318` | 高风险、极高风险提示 |
| `--info` | `#315D9C` | 置信度、说明性提示 |

风险颜色必须配合文字说明使用，不能只用颜色表达状态。低风险使用绿色系，中风险使用琥珀色，高风险和极高风险使用红棕色；极高风险不额外使用闪烁、警报等刺激性视觉。

### 字体与字号

| 场景 | 移动端 | Web/H5 宽屏 | 说明 |
|---|---:|---:|---|
| 页面标题 | 24px / 700 | 32px / 700 | 首页和报告页可使用更大标题 |
| 模块标题 | 18px / 700 | 22px / 700 | 卡片内标题保持短句 |
| 正文 | 15px / 400 | 16px / 400 | 行高 1.6，避免密集 |
| 辅助说明 | 13px / 400 | 14px / 400 | 用于隐私提示、免责声明 |
| 按钮 | 16px / 700 | 16px / 700 | 移动端按钮高度不低于 48px |
| 标签 | 12px / 700 | 12px / 700 | 风险标签、状态标签 |

字体优先使用系统字体：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif`。数字评分可使用同一字体加粗，不引入装饰字体。

### 间距与栅格

| Token | 值 | 用途 |
|---|---:|---|
| `space-1` | 4px | 标签内部间距 |
| `space-2` | 8px | 图标与文字、列表项内部 |
| `space-3` | 12px | 表单项之间 |
| `space-4` | 16px | 卡片内边距 |
| `space-5` | 24px | 模块之间 |
| `space-6` | 32px | 页面段落间距 |

移动端页面左右边距 16px，最大内容宽度 430px；Web/H5 宽屏内容区最大 1120px，使用“左侧导航 + 中间手机预览 + 右侧说明”的评审布局。所有核心页面在 360px 宽度下不得出现横向滚动。

## 页面设计说明

### 首页

首页用一句话解释产品价值：“粘贴岗位描述，识别信息不一致风险”。首屏只放一个主按钮“检测岗位风险”，两个次级入口“分析 HR 回复”“提交面试反馈”。隐私提示放在按钮下方，弱化但可见。

首页应避免用大面积恐吓文案。建议使用“少跑一次无效面试”“问清楚再决定”这类中性表达。

### 岗位检测页

检测页以大文本框为核心。平台、公司名称、岗位名称属于辅助字段，允许为空；JD 文本为必填。HR 聊天内容作为可展开输入区，避免首屏表单过长。

输入区必须放置两类提示：

- 内容提示：可粘贴岗位职责、任职要求、薪资、HR 对话。
- 隐私提示：不建议上传身份证、银行卡、完整手机号等敏感信息。

提交按钮在用户未输入 JD 时置灰；输入超过 50 字后进入可点击状态。加载状态使用进度文案“正在分析岗位标题、职责、薪资和风险话术...”，不展示虚假的百分比。

### 风险报告页

报告页首屏先展示综合风险分、风险等级、置信度和一句建议。用户不需要先读完模型解释，才能知道下一步怎么做。

推荐模块顺序：

1. 风险摘要：分数、等级、置信度、建议动作。
2. 命中风险信号：每条信号包含证据片段和解释。
3. 缺失关键信息：合同主体、固定底薪、客户来源、销售指标等。
4. 建议追问：直接进入可复制的问题列表。
5. 子分说明：JD、HR、工商、用户反馈，冷启动缺失数据标注“暂未使用”。
6. 免责声明和误判反馈入口。

报告中不得输出“确定是诈骗”“一定有问题”等法律或事实定性。高风险结论必须绑定证据片段。

### 追问卡片页

追问卡片把报告转化成可发给 HR 的话。问题数量建议 3–6 个，默认按风险影响排序。每条问题提供“复制”按钮，底部提供“复制全部”和“粘贴 HR 回复继续分析”。

卡片文案保持礼貌、具体、可被 HR 回答：

```text
您好，我想确认一下：这个岗位前 1-3 个月是否有个人销售指标？
客户来源是公司提供，还是需要自己开发？
劳动合同签署主体是哪家公司？
```

### 面试反馈页

反馈页采用分段表单，先收集公司、岗位、平台，再收集“JD 写的内容”和“面试实际说的内容”。涉及销售、收费、培训贷、押金、主体不一致等字段使用单选按钮，减少用户输入负担。

“是否匿名公开”默认关闭，提交按钮上方说明反馈会先审核，不公开聊天记录原文。

### 隐私说明与免责声明

隐私说明使用短段落和项目符号展示，避免法律文本堆叠。免责声明需要明确：风险报告仅供求职决策参考，不对企业或岗位作法律定性；用户应结合实际沟通、劳动合同和官方渠道判断。

## 交互状态

| 状态 | 页面 | 表现 |
|---|---|---|
| 默认 | 首页、检测页 | 主按钮突出，次按钮使用描边或浅色底 |
| 输入中 | 检测页、反馈页 | 字数提示、隐私提示常驻 |
| 不可提交 | 检测页 | 主按钮置灰，提示“请先粘贴岗位描述” |
| 分析中 | 检测页 | 页面禁止重复提交，显示分析文案 |
| 成功 | 报告页、反馈页 | 展示结果摘要和下一步入口 |
| 错误 | 检测页 | 说明失败原因，保留用户已输入内容 |
| 空数据 | 历史、可选模块 | 使用“暂未检测岗位”类中性文案 |

## Web/H5 兼容规则

1. 使用响应式单列内容作为基础，宽屏只增加辅助布局，不改变页面主次关系。
2. 表单控件高度不低于 44px，主按钮不低于 48px。
3. 固定底部操作区需要兼容 iOS 安全区：`padding-bottom: env(safe-area-inset-bottom)`。
4. 文本输入框在 H5 上允许自动增高，小程序上保持可滚动。
5. 所有卡片圆角统一 18–24px，避免不同端视觉割裂。
6. 报告页长内容按模块折叠，默认展开摘要、风险信号和追问问题。
7. 复制操作在 Web/H5 使用 Clipboard API；小程序使用 `wx.setClipboardData`。

## 文案边界

推荐使用：

- “信息不透明”
- “建议补问关键信息”
- “存在包装风险”
- “需要先确认合同主体”
- “仅供求职决策参考”

避免使用：

- “骗子公司”
- “诈骗”
- “黑名单”
- “曝光”
- “实锤”
- “违法”

界面文字应把判断限定在“岗位信息和沟通内容”层面，而不是给企业或个人定性。
