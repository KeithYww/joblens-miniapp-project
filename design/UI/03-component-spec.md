## 基础组件

### 顶部导航

用途：H5/Web 原型、移动 H5 顶部栏、桌面 Web 顶部导航。

组成：

- 产品名：职镜 JobLens
- 页面标题
- 返回按钮，二级页面展示
- 可选说明入口：隐私、免责声明

规范：

- 移动端高度 56px，左右边距 16px。
- 标题最多一行，超出省略。
- Web 原型可扩展为顶部导航和锚点链接。

### 底部操作栏

用途：报告页、追问卡片页、表单页的固定主操作。

组成：

- 主按钮：例如“检测岗位风险”“复制全部问题”
- 次按钮：例如“保存报告”“继续分析 HR 回复”
- 安全区留白

规范：

- 移动端固定在底部，背景使用 `color.surface`。
- 高度不低于 72px，按钮高度不低于 48px。
- 内容长页底部增加同等高度占位，避免遮挡。

### 按钮

| 类型 | 用途 | 样式 |
|---|---|---|
| 主按钮 | 页面主动作 | 品牌色填充，白字，圆角 14px |
| 次按钮 | 辅助入口 | 浅色底或描边，品牌色文字 |
| 文本按钮 | 隐私、免责声明、纠错 | 无底色，使用品牌色或辅助色 |
| 危险提示按钮 | 停止继续、删除输入 | 仅在确有删除或停止动作时使用红色 |

状态：

- 默认：可点击。
- 按下：颜色加深，轻微缩放不超过 0.98。
- 禁用：浅灰底，文字 `color.muted`，不可提交。
- 加载：按钮文字替换为加载文案，不允许重复点击。

### 输入框

用途：JD 文本、HR 聊天内容、反馈内容。

规范：

- 单行输入高度 48px。
- 多行输入最小高度 160px，JD 主输入建议 220px。
- 边框 `1px color.rule`，聚焦时使用 `color.brand`。
- 占位文案说明可粘贴内容，不使用营销语。

示例占位文案：

```text
粘贴岗位职责、任职要求、薪资范围、福利说明等内容。
```

隐私提示：

```text
不建议上传身份证、银行卡、完整手机号等敏感信息。
```

### 标签选择

用途：招聘平台、风险类型、反馈选项。

规范：

- 标签高度 34px，圆角 999px。
- 未选中使用浅色底和辅助文字。
- 选中使用品牌浅底和品牌文字。
- 单选标签可横向换行，不依赖横向滚动。

## 业务组件

### 风险摘要卡

用途：风险报告页首屏。

组成：

- 综合风险分：0–100，大号数字。
- 风险等级：低 / 中 / 高 / 极高。
- 置信度：低 / 中 / 高。
- 疑似真实岗位类型。
- 一句话建议。
- 主动作按钮：复制追问问题或生成追问卡片。

规范：

- 风险分和等级必须同时展示。
- 高风险色只用于数字、标签和重点线条，卡片背景保持白色或浅暖色。
- 置信度低时，提示“信息较少，建议补充 HR 回复或合同主体信息”。

### 风险等级 Badge

| 等级 | 分数 | 文案 | 色彩 |
|---|---:|---|---|
| 低风险 | 0–30 | 低风险，可正常沟通 | `color.low` |
| 中风险 | 31–60 | 中风险，建议补问关键信息 | `color.medium` |
| 高风险 | 61–80 | 高风险，建议先电话确认 | `color.high` |
| 极高风险 | 81–100 | 极高风险，谨慎继续 | `color.high` |

Badge 必须包含文本，不允许只用颜色或图标。

### 命中证据卡

用途：解释风险报告为什么给出该判断。

组成：

- 风险类型：如“标题职责偏差”“薪资透明度不足”。
- 证据片段：引用用户输入中的原文。
- 解释：为什么这段信息需要进一步确认。
- 追问入口：关联到一个具体追问问题。

规范：

- 证据片段使用引号或浅色引用块。
- 每条证据不超过 120 字，过长可折叠。
- 无证据不得展示高风险结论。

### 缺失信息卡

用途：提示用户需要向 HR 补问的信息。

常见缺失项：

- 固定无责底薪
- 是否有个人销售指标
- 客户来源
- 劳动合同签署主体
- 社保缴纳主体
- 是否存在培训费、押金、材料费

规范：

- 用“待确认”而不是“有问题”描述缺失项。
- 每项缺失信息可直接生成追问问题。

### 追问问题卡

用途：追问卡片页和报告页内嵌模块。

组成：

- 问题正文
- 风险关联标签
- 复制按钮
- 可选解释：为什么要问

规范：

- 每个问题保持礼貌、具体、单一问题点。
- 问题数量默认 3–6 个，最多 8 个。
- “复制全部”输出时按换行分隔。

示例：

```text
您好，我想确认一下：这个岗位前 1-3 个月是否有个人销售指标？
```

### 子分条形组件

用途：展示 JD、HR、工商、用户反馈各子模型分数。

规范：

- 以横向进度条展示，分数右对齐。
- 缺失数据显示“暂未接入”或“暂无样本”，进度条为空。
- 不把缺失项显示为 0 分，避免误导用户。

### 隐私提示条

用途：首页、检测页、反馈页、报告页底部。

规范：

- 使用浅色背景和信息色文字。
- 文案短，不超过两行。
- 提供隐私说明入口。

推荐文案：

```text
聊天记录和 JD 原文不会公开展示。请勿上传身份证、银行卡、完整手机号等敏感信息。
```

### 免责声明块

用途：报告页底部、免责声明页。

规范：

- 报告页使用短文本。
- 免责声明页使用完整说明。
- 不使用弹窗强打断用户，但首次检测前可用勾选确认。

推荐短文本：

```text
风险报告仅供求职决策参考，不对企业或岗位作法律定性。
```

## 表单组件

### 分段表单

用途：面试反馈页。

组成：

- 基本信息：公司、岗位、平台。
- JD 与实际：两个多行输入框。
- 风险事实：销售、收费、培训贷、押金、主体不一致。
- 公开设置：是否匿名公开。

规范：

- 每段使用卡片分隔。
- 单选题使用“是 / 否 / 不确定”。
- 提交前显示匿名和审核说明。

### 开关

用途：是否匿名公开。

规范：

- 默认关闭。
- 开启时说明“审核后仅展示结构化信息，不公开原文聊天记录”。

## 状态组件

### 加载状态

文案：

```text
正在分析岗位标题、职责、薪资和风险话术...
```

规范：

- 不展示虚假进度百分比。
- 分析超过 8 秒可提示“内容较长，仍在分析”。
- 保留返回和取消入口。

### 空状态

用途：检测历史、反馈记录等可选模块。

文案：

```text
暂未检测岗位。粘贴一段 JD，先看看哪些信息需要问清楚。
```

### 错误状态

规范：

- 说明原因和下一步。
- 保留用户输入内容。
- 提供“重新检测”和“复制已输入内容”。

示例：

```text
分析失败，可能是网络不稳定。你粘贴的内容已保留，可以稍后重试。
```

### Toast

用途：复制成功、提交成功、保存成功。

规范：

- 位置：移动端底部操作栏上方；Web 端右上角或屏幕底部。
- 展示时长：1.6–2 秒。
- 文案短句。

示例：

```text
已复制，可粘贴给 HR
```

## 响应式规则

| 宽度 | 布局 |
|---:|---|
| ≤ 430px | 单列移动端，固定底部操作栏 |
| 431–767px | H5 单列，内容居中，最大宽度 430px |
| 768–1199px | 原型评审布局，页面卡片两列排列 |
| ≥ 1200px | 左侧导航 + 中间页面预览 + 右侧说明 |

组件在所有宽度下都保持同一信息顺序。宽屏可以并排展示模块，但不能改变“摘要 → 证据 → 缺失信息 → 追问”的报告阅读顺序。

## 开发映射建议

| 组件 | 移动 H5 实现 | 桌面 Web 实现 |
|---|---|---|
| 顶部导航 | 自定义 navigation bar | `header` + sticky |
| 底部操作栏 | `view` fixed + safe area | `position: sticky/fixed` |
| 多行输入 | `textarea` | `textarea` |
| 标签选择 | `view` + class state | `button` 或 `label` |
| 风险摘要卡 | 业务组件 `risk-summary` | 组件 `RiskSummary` |
| 追问问题卡 | 业务组件 `question-card` | 组件 `QuestionCard` |
| Toast | 自定义 toast | 自定义 toast 或浏览器提示 |
| 复制 | `navigator.clipboard.writeText` + 手动复制兜底 | `navigator.clipboard.writeText` |

## 可访问性

1. 所有按钮文本表达动作，不使用只有图标的按钮。
2. 风险等级不能只靠颜色区分，必须包含文字。
3. 输入框需要明确 label，不只依赖 placeholder。
4. Web/H5 版本键盘可聚焦，主操作使用原生 `button`。
5. 正文和背景对比度不低于 WCAG AA。
