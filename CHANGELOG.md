# 更新日志

## 2026-07-15

### 新增：全球与中国双环境隔离发布

- 全球环境继续使用 Vercel 前端与 Render 后端，中国环境使用腾讯云上的 Caddy、Fastify、PostgreSQL 和 Redis Docker Compose 服务。
- 两套环境的数据库、Redis、报告、反馈、缓存、AI 额度和管理凭据完全隔离，避免跨境数据混用和单环境故障扩散。
- GitHub Actions 在 `main` CI 通过后发布同一个 commit SHA；全球和中国环境的前端版本、后端版本及健康检查全部一致才允许发布门禁通过。
- 中国部署使用不可变版本目录、发布前数据库备份、共享密钥目录和失败回滚，避免直接覆盖线上工作目录。

### 安全：国内 IP 入口启用可信 HTTPS

- 国内入口更新为 [https://43.142.15.246](https://43.142.15.246)，无需购买域名；HTTP 请求统一返回 308 并跳转到 HTTPS。
- 接入 Let's Encrypt 短期 IP 证书，由服务器定时任务每日检查续期，Caddy 使用续期后的证书提供服务。
- 修复无 SNI 客户端无法获得正确 IP 证书的问题，并清理历史自签名证书，避免浏览器显示不安全警告。
- 生产巡检同步检查全球与中国入口、数据库、Redis、版本号和独立监控凭据。

### 运维：双管理后台与独立凭据

- README 增加全球管理后台与中国管理后台入口，并明确两个环境的 `ADMIN_TOKEN` 不能交叉使用。
- 管理 Token 不写入仓库或前端环境变量，仅在服务端配置；浏览器登录态继续使用标签页级 `sessionStorage`。

### 修复：招聘宣传长图 OCR 识别

- OCR 提示词升级至 v2：没有具体岗位的招聘宣传海报也会提取公司介绍、价值观、发展历程和招聘背景，回填为可编辑文本。
- 招聘口号不再被误判为 HR 对话；模型返回的空岗位名称、平台和对话字段会在进入缓存前清理。
- 图片确实不含可用招聘信息时返回明确的 `422 NO_JOB_INFORMATION`，前端提示上传包含岗位职责或任职要求的截图，不再误报 `OCR_UNAVAILABLE`。
- 使用 Gate 招聘长图完成线上真实模型回归：`Qwen/Qwen3-VL-8B-Instruct` 返回 HTTP 200，并正确识别公司名称和招聘介绍。
- OCR Prompt 版本更新后自动隔离旧缓存，并新增宣传海报解析与无招聘信息场景测试。

### 修复：中英文报告与面试反馈完整适配

## 2026-07-14

### 修复：OCR 长请求卡住

- 视觉模型请求超过 60 秒时由后端主动终止，并返回明确的 `504 OCR_TIMEOUT`。
- 前端最多等待 65 秒，超时后结束进度状态并提示重试或手动填写，不再无限停留在识别中。
- 视觉模型超时支持通过 `SILICONFLOW_VISION_TIMEOUT_MS` 配置，限制范围为 10 至 90 秒。

### 调整：报告结果缓存

- 相同访客、相同岗位输入的模型结果缓存从 7 天缩短为 10 分钟。
- 报告数据保留期限不变；10 分钟后再次检测会重新调用模型。

### 修复：岗位风险评分校准

- 管理岗标题同时包含多项销售职责时，稳定识别为“管理岗包装销售岗”。
- 明确培训贷、贷款分期、扣留证件会提升为极高风险；押金、保证金、先交费会提升为高风险。
- 正常且薪资、指标透明的销售岗不会因销售属性被误判为管理岗包装。
- 信息过少且无强风险信号时降为低置信度，不作明确风险判断。
- 养老、保险、金融事业部的泛化管理/辅助岗位，如同时出现高薪低门槛与职责模糊，会提示疑似业务性质需确认，并生成保险销售、客户开发、业绩指标与合同主体追问。

### 新增：岗位截图 OCR 导入

- 首页支持上传 1 至 3 张岗位详情或 HR 聊天截图。
- 支持 PNG、JPEG、WebP，单张最大 2MB。
- 接入 SiliconFlow 视觉模型提取岗位 JD、公司名称、岗位名称、招聘平台和 HR 聊天内容。
- 识别结果会回填到表单，用户可确认、修改后再发起风险检测。
- 新增 `POST /api/ocr/extract-job` 接口。

### 隐私与稳定性

- 截图只在识别请求中转发给视觉模型，不写入数据库或业务日志。
- OCR 接口复用匿名访问标识、限流和验证码保护。
- 部署模板新增视觉模型配置与 10MB 请求体上限，支持三张图片的 Base64 请求。
