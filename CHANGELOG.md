# 更新日志

## 2026-07-15

### 新增：管理后台

- 新增经营总览、分析质量、反馈审核、系统与安全四个管理工作台。
- 支持 1/7/30 天指标、风险与模型分布、报告搜索筛选、反馈审核和安全事件查看。
- 管理接口使用独立 Admin Token，所有返回 DTO 移除 IP、visitor、user-agent 和输入哈希。

### 安全：上线前监控、备份与敏感数据加固

- 生产 Turnstile 从禁用改为 managed 模式，并校验线上 hostname。
- OCR 结果在写入跨 visitor Redis 缓存前增加后端高敏信息检测。
- 新增前端运行时错误上报、API 成功率指标和受 token 保护的内部监控接口。
- 新增每 30 分钟生产巡检及 AI 日预算 80% 告警。
- 新增 Render Free 核心业务数据每日加密快照与恢复结构校验，备份保留 14 天。
- 监控与备份 Token 拆分，备份剔除 IP 地址；生产巡检增加 Vercel 首页可用性校验。

### 优化：首页输入校验与功能入口反馈

- 岗位 JD 和 HR 聊天记录增加手机号、身份证号、银行卡号实时检测，提交前提示用户删除或打码。
- 文本接近 8000 字时显示警告色，超限时显示需删减字数并阻止提交。
- “风险识别”点击后高亮并聚焦岗位输入区，同时展示操作提示。
- “追问建议”优先跳转最近报告的追问区域；无报告时明确提示先完成岗位检测。

### 优化：首页品牌与功能导航

- 浏览器标题更新为“职镜 JobLens - 岗位风险智能检测”，并补充中文页面语言和搜索描述。
- 首页底部“风险识别”可返回检测表单，“追问建议”可打开最近报告，“面试反馈”可进入匿名反馈页面。
- 三个功能入口补充键盘焦点、移动端单列布局和明确的交互状态。

### 新增：AI 成本控制 P0

- 匿名用户每天最多使用 3 次 AI OCR 和 3 次 AI 文本分析，并增加 IP 聚合额度。
- 使用 Redis 原子脚本限制每日 300 个全站积分，OCR 每次计 3 分、文本分析每次计 1 分。
- 全站模型调用最多并发 4 个，其中 OCR 最多 2 个、文本分析最多 3 个。
- 相同截图按图片内容、语言、模型和 Prompt 版本跨 visitor 缓存 24 小时，缓存命中不扣 AI 额度。
- 预算耗尽或 Redis 不可用时，文本分析明确降级为基础规则报告；OCR 停止付费调用并提示手动填写。
- 首页展示当前 visitor 的 OCR 与分析剩余次数，报告页明确标识基础规则报告。
- 新增真实 Redis 集成测试，验证额度预占、退款、全站预算和并发租约。

## 2026-07-14

### 修复：OCR 长请求卡住

- 视觉模型请求超过 60 秒时由后端主动终止，并返回明确的 `504 OCR_TIMEOUT`。
- 前端最多等待 65 秒，超时后结束进度状态并提示重试或手动填写，不再无限停留在识别中。
- 视觉模型超时支持通过 `SILICONFLOW_VISION_TIMEOUT_MS` 配置，限制范围为 10 至 90 秒。

### 调整：报告结果缓存

- 相同访客、相同岗位输入的模型结果缓存从 7 天缩短为 10 分钟。
- 报告数据保留期限不变；10 分钟后再次检测会重新调用模型。

### 修复：岗位风险评分校准

- 管理岗标题同时包含多项销售职责时，稳定识别为“管理岗包装销售岗”。
- 明确培训贷、贷款分期、扣留证件会提升为极高风险；押金、保证金、先交费会提升为高风险。
- 正常且薪资、指标透明的销售岗不会因销售属性被误判为管理岗包装。
- 信息过少且无强风险信号时降为低置信度，不作明确风险判断。
- 养老、保险、金融事业部的泛化管理/辅助岗位，如同时出现高薪低门槛与职责模糊，会提示疑似业务性质需确认，并生成保险销售、客户开发、业绩指标与合同主体追问。

### 新增：岗位截图 OCR 导入

- 首页支持上传 1 至 3 张岗位详情或 HR 聊天截图。
- 支持 PNG、JPEG、WebP，单张最大 2MB。
- 接入 SiliconFlow 视觉模型提取岗位 JD、公司名称、岗位名称、招聘平台和 HR 聊天内容。
- 识别结果会回填到表单，用户可确认、修改后再发起风险检测。
- 新增 `POST /api/ocr/extract-job` 接口。

### 隐私与稳定性

- 截图只在识别请求中转发给视觉模型，不写入数据库或业务日志。
- OCR 接口复用匿名访问标识、限流和验证码保护。
- 部署模板新增视觉模型配置与 10MB 请求体上限，支持三张图片的 Base64 请求。
